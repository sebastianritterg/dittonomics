# Response Protocol

Use this protocol when converting the roadmap into an actual response letter.

## Core Rules

- match the user's voice
- never fabricate results, checks, or resolved decisions
- every reviewer request must map to a task, response stance, or explicit rationale for non-compliance
- disagreements and partial concessions require user-visible review

## Response Stances

- `accepted`
- `clarified`
- `reframed`
- `partially accepted`
- `respectfully declined`
- `pending analysis`

## Diplomatic Disagreement Pattern

When the stance is not full acceptance:

1. acknowledge the reviewer concern
2. state what was checked or reconsidered
3. explain the chosen action
4. offer a partial concession when useful
5. avoid adversarial language

Never write:

- "the reviewer is wrong"
- "we refuse"
- "this comment is invalid"

## Minimum Response Matrix Columns

- reviewer source
- quote
- linked task id
- stance
- manuscript change or planned change
- location in manuscript
- note if user approval is still needed
