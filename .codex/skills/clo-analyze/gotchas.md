# Analyze Gotchas

- Do not code before a usable strategy memo exists unless the user explicitly asks for exploration.
- Resolve repo paths before assuming `paper/`, `scripts/`, or `data/`.
- Keep analysis outputs reproducible and writer-ready; `results_summary.md` is the handoff artifact.
- Run `$clo-research-tools lint` before code review when scripts changed.
- Stata, Python, R, and Julia are supported, but repo guidance decides the order.
