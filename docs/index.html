Dittonomics

A remixed Codex research workflow built from multiple upstream repos

Dittonomics is a Codex-native research workflow that openly borrows from multiple upstream repos instead of pretending it was invented from scratch.

The name is deliberate: like Ditto, this project copies useful traits, reshapes them, and turns them into one portable operating model for empirical work in Codex.

WarningUnder Active Construction

This manual and site are being updated actively. Expect changes, expansions, and new sections, especially around project-management skills, workflow options, and broader Codex integrations.

Built From

Dittonomics builds its architecture, workflow method, and prompting style from:

  • Clo-Author by Hugo Sant’Anna
  • my_claude_skills by Daria M.
  • claudeblattman by Chris Blattman

This is a remix on purpose. The project is called Dittonomics because it copies patterns from different repos and adapts them into one Codex-facing research system.

The goal is simple: preserve the workflow logic that makes the upstream systems useful for research, while translating the active instructions into Codex-native primitives:

  • AGENTS.md and AGENTS.override.md instead of CLAUDE.md
  • active workflow references instead of hidden .claude assumptions
  • explicit $clo-* skills instead of slash-command dependence
  • Codex custom subagents instead of Claude agent markdown files
  • explicit exploration scaffolding instead of undocumented folder conventions
  • a forkable .codex/ starter bundle instead of asking every user to reconstruct the user-level layer from scratch

What You Get

  • a five-phase research pipeline
  • worker-critic separation across the whole workflow
  • journal-aware peer review via editor plus blind referees
  • a Codex migration layer that makes the upstream material readable without constant translation
  • an example overlay that shows how one concrete installation and research setup can adapt the generic workflow
  • a public .codex/ starter kit that can be copied or adapted after forking

Get Started in 5 Minutes

Step 1: Understand the starter model

This repo ships a forkable starter layer under .codex/ with:

  • AGENTS.md
  • agents/
  • skills/
  • WORKFLOW_QUICK_REF.md

When you adopt it for your own machine, the main user-level Codex storage layers are:

  • user skills in ~/.codex/skills
  • custom subagents in ~/.codex/agents

Repo-specific overrides still belong in:

  • AGENTS.md
  • AGENTS.override.md
  • .agents/skills/

Step 2: Translate the upstream mental model

If the upstream manuals say:

  • CLAUDE.md -> read or create AGENTS.md
  • .claude/rules -> use active workflow references plus repo overlay guidance
  • /discover -> use $clo-discover
  • .claude/agents -> use Codex custom subagents

The public migration cheatsheet lives in Reference.

Step 3: Create or adapt a repo

For a new research repo, the minimum Codex-facing structure is:

  • root AGENTS.md
  • optional AGENTS.override.md
  • repo-local skills in .agents/skills/
  • canonical manuscript, code, output, and docs directories
  • explorations/ when the project needs a sandbox for prototypes

Step 4: Use the workflow phases

The workflow stays the same:

  1. Discovery
  2. Strategy
  3. Execution
  4. Peer Review
  5. Submission

Talks can run in parallel with review.

Step 5: Start with explicit skills

The safest Codex-native entry points are:

  • $clo-discover
  • $clo-strategize
  • $clo-analyze
  • $clo-write
  • $clo-review
  • $clo-submit

The architecture page explains how these map to agents and quality gates.

Prerequisites

Tool Why it matters Notes
Codex Main orchestrator Desktop or terminal usage both fit the architecture
Python Default automation and analysis language Primary default in this adaptation
Stata Econometric co-default Use when the repo or replication target centers Stata
XeLaTeX Paper compilation Needed for LaTeX manuscripts and slides
Git Versioning and review workflow Required for repo-based operation
Quarto Docs site only Needed to render this manual locally

R and Julia remain supported, but they are no longer the default recommendation in the active Codex adaptation.

What This Site Is Not

This site is not:

  • a literal clone of any one upstream repo
  • a promise that Codex supports hidden .claude mechanics
  • a packaged installer for one exact workstation

It is a public operating manual for a Codex-native remix of several upstream research-workflow systems.

Attribution
