Reference

The cheat sheet for skills, utilities, folders, and recurring prompts

Daily Cheat Sheet

Start the day

Use $clo-research-tools resume-context for this repo.

End the day

Use $clo-research-tools checkpoint for this repo.

After a risky edit

Use $clo-research-tools verify-edit 03_code/src/estimate.py

Workflow Skills

Skill Use it for Example
$clo-new-project launch a whole project workflow Use $clo-new-project on this topic and scaffold the pipeline.
$clo-discover interview, literature, or data discovery Use $clo-discover lit on school assignment and teacher quality.
$clo-strategize identification design and PAPs Use $clo-strategize on this question and rank the causal designs.
$clo-analyze coding, outputs, cleaning, estimation Use $clo-analyze and build the estimation pipeline plus output handoff.
$clo-write manuscript drafting, revising, humanizing, style-guide work Use $clo-write intro with current-draft mode.
$clo-review writing review, peer review, or critic workflows Use $clo-review --peer on the current paper draft.
$clo-revise response-to-referees and revision routing Use $clo-revise on this referee report.
$clo-talk deck building Use $clo-talk seminar from the current paper.
$clo-submit final submission checks Use $clo-submit and tell me if this is actually submission-ready.
$clo-research-tools utilities and maintenance Use $clo-research-tools checkpoint for this repo.

Research Tools Cheat Sheet

Utility What it does Typical use
checkpoint saves end-of-day or handoff state when stopping work
resume-context reconstructs last known working context when reopening a project
verify-edit manual post-edit safety pass after touching code or build files
context quick session-status summary when you feel lost mid-session
compile build paper or talk outputs after manuscript edits
validate-bib catch citation drift before review or submission
journal summarize research log when documenting progress
learn draft reusable learning prompts when a workflow pattern is worth saving
upgrade update infrastructure, not research content when syncing the starter or user install

Checkpoint File Format

Default structure:

.codex-state/
  checkpoints/
    2026-04-22T18-30-00Z_end-of-day.json
    2026-04-22T18-30-00Z_end-of-day.md
  latest.json
  latest.md
  resume-context.md

Use .json as the durable machine-readable layer and .md as the readable handoff layer.

Recurring Prompt Patterns

Discovery

Use $clo-discover lit on [topic]. Start repo-bibliography-first and then broaden if the frontier is moving.

Strategy

Use $clo-strategize on [question]. Give me the main design, the threats, and the robustness plan.

Writing

Use $clo-write [section] with [current-draft/joint-paper/journal-tightening] mode.

Review

Use $clo-review on the latest draft and give findings first.

End-of-day

Use $clo-research-tools checkpoint for this repo. Capture today's goal, completed work, blockers, important files, and next step.

Next morning

Use $clo-research-tools resume-context for this repo and tell me where we left off and what to do next.
